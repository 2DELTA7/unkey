---
title: "Get Key"
description: "Retrieve a key by its ID"
api: "GET /v1/keys.getKey"
authMethod: "bearer"
---

Retrieve a key to get its current config and state.

## Request

<ParamField query="keyId" type="string" required>
  The ID of the key you want to retrieve.
</ParamField>


## Response

<ResponseField name="id" type="string" required>
  The unique ID of the key. This is not the key itself and can be stored or shared safely.
</ResponseField>


<ResponseField name="apiId" type="string" required>
 The unique ID of the API this key is for.
</ResponseField>


<ResponseField name="workspaceId" type="string" required>
 The unique ID of the workspace this key is for.
</ResponseField>

<ResponseField name="start" type="string" required>
  The first few characters of the key to visually match it.
</ResponseField>

<ResponseField name="name" type="string">
  Update the name of the key.
</ResponseField>

<ResponseField name="ownerId" type="string">
  Update the owner id of the key.
</ResponseField>

<ResponseField name="meta" type="JSON">
  Update the metadata of a key. You will have to provide the full metadata
  object, not just the fields you want to update.
</ResponseField>
<ResponseField name="createdAt" type="int" requried>
  A unix timestamp in milliseconds of when the key was created.
</ResponseField>
<ResponseField name="expires" type="int">
  Update the expire time of a key.

The expire time is a unix timestamp in milliseconds.

</ResponseField>

<ResponseField name="ratelimit" type="Object" >

Unkey comes with per-key ratelimiting out of the box.

  <Expandable title="properties">

  <ResponseField name="type" type="string" required>
  Either `fast` or `consistent`.

Read more [here](/features/ratelimiting)

  </ResponseField>
  <ResponseField name="limit" type="int" required>
  The total amount of burstable requests.

  </ResponseField>
  <ResponseField name="refillRate" type="int" required>
  How many tokens to refill during each `refillInterval`
  </ResponseField>
  <ResponseField name="refillInterval" type="int" required>
  Determines the speed at which tokens are refilled.

In milliseconds

  </ResponseField>
 </Expandable>
</ResponseField>

<ResponseField name="remaining" type="int">
  How many verifications are left for this key before it is invalidated.
</ResponseField>



<RequestExample>

```sh
curl --request POST \
  --url https://api.unkey.dev/v1/keys.getKey?keyId=<KEY_ID> \
  --header 'Authorization: Bearer <UNKEY>'
```

</RequestExample>
